synsets_add_synset(self, sy_id, synset_provenance, definition, rels)

source code 

synset is added if it has a hypernym relation to an existing synset.

WARNING not added if: (1) sy_id already exists (2) no succesful hypernym relation added (except for adjectives)

Parameters:
  • sy_id (str) - synset identifier
  • synset_provenance (str) - origin english synset: 'pwn', else 'odwn'
  • definition (str) - definition
  • rels (list) - list of tuples (reltype,target)
Returns:
tuple

synsets_remove_synset(self, sy_identifier, remove_les=True, synset_el=None)

source code 

(1) if removes_les: all lexical entries are removed from this synset (2) if no hyponyms: remove synset (3) if no hyponyms: remove all relations pointing to this synset

Parameters:
  • sy_identifier (str) - synset identifier (i.e. eng-30-89405202-n)
  • remove_les (bool) - remove les in synset (default is True)
  • synset_el (None | etree.Element) - default is None, odwn will be searched for element. else, an Synset instance has to be provided

validate_relation(self, source, reltype, target)

source code 

this method check if a relation is valid or not. invalid if: (1) target does not exist (2) source does not exist (3) cross pos for has_hyponym has_hyperonym (4) source == target (5) reltype not in existing list of reltypes (6) relation exists already

Parameters:
  • source (str) - source synset identifier
  • reltype (str) - relation type
  • target (str) - target synset identifier
Returns: tuple
(succes,message)
